# 旅游AI产品分析
为了能够更好的规划WorldTravel项目，并且探索可行的结合AI的功能，我们尝试通过分析市面上主流的旅游AI产品，并总结了他们的核心功能、技术应用、用户体验、商业模式、SWOT分析等来寻找灵感。

虽然有关旅行的大模型被提到了很多次，但旅行公司一般都是在自己数据的基础上训练，作为一个局外人很难接触到类似的数据。所以在考虑功能时，更多的是尝试能力的组合和打通，至于模型检索的结果优化还需要进一步探索。

浏览了目前市面上主要的AI旅游产品大概可以分为以下几类：
1. 文字类旅行计划：通过给AI赋予提示词，基于AI本身的知识能力，可能还有网页检索能力，给与行程安排的文字规划；
2. 图文结合类计划：这一类产品通畅已经具备初步的agent能力，通过自然语言的输入，调动agent查阅网络信息，比如机票、酒店、景点、餐饮等，并且获取相应的地理信息，结合开源地图资源，通常是谷歌地图等进行展示。
3. 前两类通常自身不具备旅游咨询资源数据，第三类主要是旅行类公司，结合自身产品数据，开发垂直类模型产品。但是这个逻辑会有点问题，它提供的是一个更易于查询、规划的工具，但这不契合在平台发布信息的旅行社，会有倾向寻找低价产品？
4. 第四类是一些智能客服，或者能够自动检索、订票的Agent工具，意在解决一些相对流程化可标准化的工作。

# 竞品功能分析

各类产品使用下来有一个共同的感受。旅行是一个非常复杂的事情，需要考虑的因素非常多，而且每个人的旅行目的、旅行方式、旅行时间、旅行预算、旅行偏好都不一样。所以很难有一个产品能够满足所有人的需求。
- 最大一个感受是颗粒度不够，得到的行程通常模式是半天一个景点-吃饭-一个景点-吃饭-酒店。虽然有些做的好的产品可以进一步对话去深化某一天的的Plan，但这就又变成了AI搭了一个框架，用户去填充内容。
- 通常情况除了机票、酒店有一个平台集合，旅行涉及的餐饮、小交通、攻略、甚至住宿、签证、租车、导航等分布在不同的平台。所谓的可以检索的平台也只是打通了像google、amadas等类似咨询平台。各个软件平台也会有自己的防御，不直接暴露数据。最核心的资源。这就导致你最顺手的咨询组合是打不通的，比如小红书攻略+谷歌地图定位+airbnb酒店预订+低价机票检索+类似大众点评的餐饮检索+notion攻略笔记+一个易于分享的行程博客。

## 信息源
### 机票
[Amadeus API](https://developers.amadeus.com/)
- 提供全球航班搜索、预订、价格比较等功能
- 需要申请API key，有免费和付费版本
- 数据覆盖范围广,实时性好

[Skyscanner API](https://partners.skyscanner.net/)
- 提供航班搜索、价格比较功能
- 需要申请成为合作伙伴
- 数据来源于多个OTA和航空公司

[Kiwi.com API](https://tequila.kiwi.com/)
- 提供航班搜索、组合航线等功能
- 需要注册获取API key
- 特色是可以搜索组合航线,价格较优

[Google Flights API](https://developers.google.com/qpx-express)
- 提供航班搜索功能
- 需要Google Cloud账号和API key
- 数据准确性高,但价格较贵

[AviationStack API](https://aviationstack.com/)
- 提供实时航班数据、航班状态等信息
- 有免费和付费版本
- 主要用于航班追踪,不提供票价查询

# 产品功能构思
基于以上的分析，目前要打造一款绝对智能的旅行助手还有非常多要打通的问题。即使实现了自动化，也很难做到完全的个性化。
所以我觉得旅行AI Blog可以分成以下几个步骤走：
1. 首先做最基础的，用一个图文并茂的模式汇总各类旅行相关的信息，比如自己想去国度的攻略、签证的办理方法、资讯集合等等；
2. 整合一系列工具，包括AI的助手、适当的智能地图助手、天气的资讯、汇率的资讯等等；
3. 做一些资源的整合，包括有趣的社区、一些国家自己的旅行官网、视频的入口；

反过来看，能不能提供一个逆向的工具，结合地图、预算助手、文档助手、行程助手的旅行助手平台。这就以为这需要一个数据库。
## 地图模块
- 根据我自然语言录入的景点，实现路径地图的规划表示。（通过自然语言输入，ai解析给出对应数据存入kv，然后地图基于kv读取显示。）
- 有一个增加行程点的插件，这里通过简单的数据输入，比如地点检索+时间+费用+备注，就可在地图上增加行程点，在预算里增加行程预算。
## 表格模块
行程的规划会涉及非常多表格的功能需求，比如行程计划、预算、景点的信息等等。要处理两个场景，一个是在文档编辑时候的被动添加，一个是实时的读取数据显示（这个就类似一个组件，插入在预算模块，一旦有数据变化，就实时更新）。
- 创建一个表格工具，可以编辑、显示具体金额。（https://vxetable.cn/#/demo/realTime）
## 旅行手册模板
这是一个相对通用的模板，在这里可以定义整篇blog展示的内容。可以包括：
- 旅行的概述
- 旅行的行程
- 旅行的预算
- 预订的情况
- 旅行的攻略
- Tips
- 等等
